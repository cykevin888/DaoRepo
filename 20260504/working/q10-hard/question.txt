In an organization, `N` employees with employee IDs from `1` to `N`
work in different teams. Each employee is assigned an integer `X`
representing the employee's efficiency.

The sum of efficiencies of all employees indicates the reputation of
the organization.

Edwin is appointed manager of the organization for `Q` days. Edwin,
being short-tempered, fires one employee each day. Because the team
members have a close relationship, `K` colleagues of the fired employee
resign in protest.

These `K` colleagues are the remaining team members with the least
efficiency.

Kevin is the head of the database management system and has to update
the reputation of the organization at the end of each day. Write an
algorithm to determine the reputation of the organization at the end of
each day for `Q` days.

Input

1. The first line contains an integer `num`, representing the number of
   employees in the efficiency list. (`num` is always equal to the
   given number of employees `N`.)

2. The second line contains `N` space-separated integers:
   `eff0 eff1 ... effN-1`, representing the efficiencies of the
   employees.

3. The third line contains an integer `numT`, representing the number
   of employees with a team ID. (`numT` is always equal to the given
   number of employees `N`.)

4. The fourth line contains `N` space-separated integers:
   `idE0 idE1 ... idE[N-1]`, representing the team ID of each employee.

5. The fifth line contains two space-separated integers:
   `numDays` and `numE`, representing:
   - the number of days for which Edwin is manager (`Q`)
   - the number of elements in each of the next `Q` lines
   (`numE` / `P` is always equal to `2`)

6. The last `Q` lines each contain two space-separated integers:
   `idFire` and `numResign`, representing:
   - the ID of the employee fired on that day
   - the number of employees who resign along with the fired employee
     (`K`)

Output

Print `Q` space-separated integers representing the reputation of the
organization at the end of each day.

Constraints

The original OCR text of this part is unclear. The recoverable content
is:

- `num = numT`
- `1 <= numDays <= num`
- `1 <= idFire <= num`
- `0 <= numResign <= c`, where `c` is the number of employees in the
  team on the current day
- `1 <= idE0, idE1, ...`

The following lines in the source appear truncated / ambiguous:

- `1s nums 10`
- `-10 s eff0, eff1, ... effN-1 <= 10^9`
- `... idE[N-1] ... numT`

Note

The ID of a fired employee cannot be the ID of a person who has already
resigned.

Example

Input:
5
1 2 3 4 5
5
1 2 1 1 2
2 2
3 2
2 0

Output:
7 5

Explanation:
There are two teams (IDs `1` and `2`).

The initial reputation of the organization is:
`1 + 2 + 3 + 4 + 5 = 15`

At the end of day 1, employee `3` is fired. Employee `1` and employee
`4` resign. The revised reputation is:
`7 = 15 - 3 - 1 - 4`

At the end of day 2, employee `2` is fired but nobody resigns. The
revised reputation is:
`5 = 7 - 2`

这 `K` 名辞职员工，是该团队中在当天仍然留下的成员里效率值最低的 `K`

Kevin 负责数据库管理系统，他需要在每天结束时更新组织的声誉值。请设计算法，
求出连续 `Q` 天中每天结束后的组织声誉值。

样例解释

`1 + 2 + 3 + 4 + 5 = 15`
